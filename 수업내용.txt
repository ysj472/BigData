"빅데이터 시스템 수업에서는 파이썬 기초 문법부터 시작해서,
공공데이터 API나 네이버 API 같은 걸 이용한 웹 크롤링, Selenium 라이브러리를 이용한 자동화 수집을 실습했고,
통계 분석, 텍스트 빈도 분석, 지리정보 분석 같은 기법도 다뤘어.
후반에는 KoGPT 모델을 활용해서 챗봇을 구현하고, Flask로 웹서버도 직접 구성해서 대화형 시스템을 만들었어.
정리하면 '수집 → 정제 → 분석 → 시각화 → AI 활용'까지 데이터 분석의 전 과정을 실제로 해본 수업이었어."





### 🔹 실제 경험/깊이 묻기

> **Q. 웹 크롤링은 어떤 데이터를 수집했어?**
> A. 네이버 뉴스 API를 이용해서 키워드 기반 뉴스 기사 데이터를 수집했고, 공공데이터포털에서는 관광 통계나 보건 기관 현황 같은 공공 데이터를 받아봤어요. 또, Selenium으로는 로그인이나 동적 로딩이 필요한 웹페이지에서 뉴스 헤드라인이나 댓글도 수집해봤어요.

> **Q. Selenium이랑 API 방식 중 어떤 게 더 편했어?**
> A. API 방식이 안정적이고 구조화된 데이터라 다루기가 훨씬 쉬웠어요. 대신 API가 없는 경우에는 Selenium을 써야 했는데, 그건 오류 처리나 딜레이 설정 같은 부분이 좀 더 복잡하긴 했어요.

> **Q. KoGPT로 만든 챗봇은 어떤 기능을 했어?**
> A. 사용자가 입력한 질문을 KoGPT에 전달해서 응답을 받아오는 기본 구조였어요. 예를 들어 “빅데이터란?” 같은 질문에 자연스러운 설명을 해주거나, 간단한 잡담도 가능했어요. Colab에서 미세하게 학습시키기도 했고요.

> **Q. Flask 웹서버는 직접 배포도 해봤어?**
> A. 로컬에서는 Flask로 실행해서 웹 브라우저에서 테스트했어요. 실제 클라우드 배포까지는 안 했지만, 로컬 기반 라우팅, API 통신, HTML 템플릿 연동은 직접 구현했어요.

> **Q. 데이터 시각화는 어떤 식으로 보여줬어? 지도도 썼어?**
> A. 네, folium을 사용해서 지도를 만들었고, 커피숍 주소 데이터를 지도에 마커로 찍어보는 실습을 했어요. 또 matplotlib이나 seaborn으로는 텍스트 빈도 분석 결과를 워드클라우드나 막대그래프로 시각화했어요.

> **Q. 통계 분석은 어떤 데이터를 가지고 했어?**
> A. 와인 품질 데이터셋을 이용했어요. 레드와 화이트 와인의 품질 분포를 비교하고, t-검정이나 회귀 분석으로 품질에 영향을 주는 요인을 분석해봤어요.

### 🔹 기술 스택 연관

> **Q. Node.js도 썼다고 했는데, 어디에 썼어?**
> A. Node.js는 Flask랑 연동해서 API 게이트웨이 역할로 썼어요. 챗봇 입력을 중간에서 받아서 Flask로 넘기고 응답을 다시 받아서 처리하는 방식이었어요. Express 프레임워크를 기본으로 구성했어요.

> **Q. KoGPT는 직접 학습시킨 거야? 아니면 API만 써본 거야?**
> A. 둘 다 경험했어요. 초반엔 API로 사용하다가, 나중엔 Colab에서 데이터셋을 만들고 Fine-tuning해서 성능을 비교해봤어요. 한국어 기반이라 일상 대화엔 꽤 자연스러웠어요.

> **Q. MySQL도 연동해봤다고 했는데, 어떤 식으로 저장했어?**
> A. 사용자의 질문과 KoGPT 응답을 MySQL에 자동 저장하는 기능을 Flask에서 구현했어요. 대화 로그를 저장해서 이후에 챗봇 학습용으로도 쓸 수 있도록 했어요.

### 🔹 직무 연관 또는 역량 확인

> **Q. 그럼 너는 데이터 분석이랑 개발 중에 어느 쪽이 더 관심 있어?**
> A. 저는 데이터를 활용해서 의미 있는 결과를 도출하는 걸 좋아해서 **데이터 분석 쪽에 좀 더 흥미가 있어요.** 하지만 그걸 구현하기 위한 웹개발이나 자동화 과정도 같이 배우는 걸 즐깁니다.

> **Q. 이 수업에서 배운 걸로 뭔가 프로젝트 해본 거 있어?**
> A. 수업 내용 기반으로 뉴스 키워드 분석 시스템을 만들어봤어요. 특정 키워드로 뉴스 기사 수집 → 형태소 분석 → 빈도 분석 → 워드클라우드로 시각화까지 자동으로 되는 구조였어요.

> **Q. RAG 같은 것도 써봤어? 문서 기반 응답이랑 어떤 차이 있어?**
> A. 네, RAG도 해봤어요. 챗봇 비서는 사전 학습된 모델로만 답하지만, RAG는 외부 문서를 벡터화해서 검색한 뒤 그 내용을 바탕으로 GPT가 답해요. 예를 들어, 회사 메뉴얼이나 정책을 넣으면 정확한 응답을 줄 수 있어서 실제 업무에 훨씬 유용해요.

### 🔹 응용/확장형 질문

> **Q. KoGPT 챗봇을 상용 서비스로 쓸 수 있을까?**
> A. 기본적인 응답에는 사용할 수 있지만, 정확성과 맥락 기억은 한계가 있어서 실제 서비스용으로 쓰려면 사용자 질문 로그를 학습시키거나 RAG 구조로 보완해야 해요. 다만 한국어 응답 성능은 꽤 좋았어요.

> **Q. 이걸 졸업작품이나 포트폴리오로 쓸 수 있을까?**
> A. 네, 충분히 가능하다고 생각해요. 데이터 수집부터 분석, 시각화, AI 응답까지 전 과정을 실습해서 포트폴리오로 보여주기 좋고, 특히 Flask로 웹까지 연결한 구조라 완성도도 높다고 생각해요.
